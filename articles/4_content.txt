用户想要一个**个人博客**，要求：

- 简洁、有设计感，不要千篇一律的 Hexo 模板
- 技术栈尽量简单，**不用 Hexo/Hugo 这类静态网站生成器**
- 要能部署到 GitHub Pages
- 文章用 Markdown 写，方便维护

### 第一步：写设计规范（SPEC.md）

在没有写任何代码之前，先把设计规范写清楚：

- **设计风格**：Notion 克制美学 + Hacker News 信息密度 + 东方留白感
- **配色系统**：浅色/深色两套，CSS 变量控制
- **字体**：Noto Serif SC（标题） + Inter（正文） + JetBrains Mono（代码）
- **布局**：首页卡片列表 + 文章页双栏（正文 + 目录）

### 第二步：搭前端界面

核心文件：

- `index.html` — 首页，文章卡片列表
- `article.html` — 文章页，支持 Markdown 渲染和目录生成
- `about.html` — 关于页
- `css/style.css` — 主样式，全站共享
- `js/app.js` — 核心逻辑

### 第三步：加文章

预置了多篇示例文章（`articles/1.md`、`articles/2.md`、`articles/3.md`、`articles/4.md`），每篇都有完整的 YAML front-matter。

文章索引在 `articles/index.json`，前端加载时读取这个文件展示文章列表。

### 第四步：配置 GitHub Pages 部署

配了 GitHub Actions 工作流自动部署到 GitHub Pages。

### 问题一：上下文丢失

跑了大约 19 小时后，微信会话被自动重置了，上下文全部丢失。

**教训**：长时间任务要及时保存中间状态到文件，不能依赖会话的连续性。

### 问题二：GitHub 授权

AI 助手无法直接操作用户的 GitHub 账号，需要用户自己配置 GitHub Personal Access Token。

### 问题三：深色模式调试

配色方案改了十几版才满意，因为要保证每种颜色在浅色/深色两种模式下都协调。

用 CSS 变量统一管理颜色，浅色模式一套变量，深色模式一套变量。

最终成果
```

- **博客源码**：纯 HTML + CSS + JS，不依赖任何框架
- **部署方式**：GitHub Actions 自动部署到 GitHub Pages
- **文章格式**：Markdown + YAML front-matter
- **主题**：深色/浅色模式切换
- **大小**：CSS + JS + HTML 加起来不到 50KB

反思
```

1. **先写规范再写代码**这件事比想象中重要
2. **工具没有最好的，只有最适合的**
3. **AI 助手的上下文有限**，长时间任务需要及时把进展存到文件里
