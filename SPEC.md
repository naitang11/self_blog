# 个人博客 - 设计规范 (SPEC.md)

## 1. 概念与愿景

一个**有灵魂的极客风个人博客**——不是那种千篇一律的 Hexo 模板，也不是 Medium 的复刻。灵感来源：Notion 的克制美学 + Hacker News 的信息密度 + 一丝东方留白感。读起来像一本精心排版的杂志，用起来像一个趁手的工具。

**核心感受**：安静、专注、有品味。

## 2. 设计语言

### 色彩系统

**浅色主题 (Light)**
| 角色 | 色值 | 用途 |
|------|------|------|
| 背景 | `#FAFBFC` | 页面底色 |
| 卡片 | `#FFFFFF` | 内容块 |
| 边框 | `#E8ECF0` | 分割线 |
| 主文字 | `#1A1D21` | 正文 |
| 次文字 | `#6B7280` | 元信息 |
| 强调色 | `#2563EB` | 链接、按钮、标签 |
| 标签色 | `#EFF6FF` + `#2563EB` | 标签背景/文字 |

**深色主题 (Dark)**
| 角色 | 色值 | 用途 |
|------|------|------|
| 背景 | `#0F1117` | 页面底色 |
| 卡片 | `#1A1D27` | 内容块 |
| 边框 | `#2D3344` | 分割线 |
| 主文字 | `#E8ECF0` | 正文 |
| 次文字 | `#8B95A5` | 元信息 |
| 强调色 | `#60A5FA` | 链接、按钮、标签 |
| 标签色 | `#1E3A5F` + `#60A5FA` | 标签背景/文字 |

### 字体
- **标题**: `"Noto Serif SC", Georgia, serif` — 有书卷气，中英文兼顾
- **正文**: `"Inter", "Noto Sans SC", system-ui, sans-serif` — 清晰现代
- **代码**: `"JetBrains Mono", "Fira Code", monospace` — 等宽，程序员标配
- 字号体系：12 / 14 / 16 / 20 / 24 / 32 / 48px

### 间距
基于 8px 网格系统。常用：`8 / 16 / 24 / 32 / 48 / 64 / 96px`

### 动效哲学
- 克制：不炫技，动效是调味剂，不是主菜
- 快速：过渡 150-300ms，使用 `ease-out`
- 有意义：hover 状态需要反馈，页面切换需要过渡

### 视觉元素
- 图标：Lucide Icons（线性风格，SVG）
- 圆角：8px（卡片）、4px（按钮/标签）、2px（代码块）
- 阴影：仅在深色模式使用微妙阴影，浅色模式用边框代替

## 3. 布局与结构

### 首页 (`index.html`)
```
┌──────────────────────────────────────────────┐
│  [Logo/名字]          [搜索] [主题切换] [关于] │  ← 顶部导航栏（固定）
├──────────────────────────────────────────────┤
│                                              │
│  欢迎语 / 个人标语（大字，居中）                │  ← Hero 区域
│                                              │
├──────────────────────────────────────────────┤
│  [标签筛选: 全部 | 技术 | 生活 | 随笔 ...]     │  ← 标签导航
├──────────────────────────────────────────────┤
│  ┌────────────────────────────────────────┐  │
│  │ 文章卡片（封面图 + 标题 + 摘要 + 元信息）  │  │
│  └────────────────────────────────────────┘  │
│  ┌────────────────────────────────────────┐  │
│  │ 文章卡片                                │  │
│  └────────────────────────────────────────┘  │
│  ...                                         │
└──────────────────────────────────────────────┘
```

### 文章页 (`article.html`)
```
┌──────────────────────────────────────────────┐
│  [Logo]        [搜索] [主题] [←返回]          │
├──────────────────────┬───────────────────────┤
│                      │                       │
│  标题（大字）          │   目录（sticky）       │
│  元信息（日期/分类/阅读时间）│   - 第1章           │
│                      │   - 第2章             │
│  封面图               │   - 第3章             │
│                      │                       │
│  正文内容             │                       │
│  Markdown 渲染        │                       │
│                      │                       │
│  ─────────────────   │                       │
│  评论区               │                       │
│                      │                       │
└──────────────────────┴───────────────────────┘
```

### 关于页 (`about.html`)
简洁的单栏布局：头像 + 个人介绍 + 技能标签 + 社交链接

### 响应式策略
- **桌面 (>1024px)**: 双栏（正文 + 目录）
- **平板 (768-1024px)**: 单栏，目录折叠到顶部
- **手机 (<768px)**: 全宽堆叠，汉堡菜单

## 4. 功能清单

### 核心功能
- [ ] **文章列表**：动态加载，支持按标签筛选
- [ ] **文章阅读**：Markdown 渲染，代码高亮（highlight.js），阅读时间计算
- [ ] **全文搜索**：按标题 + 内容搜索，实时过滤
- [ ] **标签系统**：多标签，支持 OR/AND 筛选
- [ ] **深色/浅色切换**：localStorage 持久化，跟随系统偏好
- [ ] **目录导航**：根据 Markdown H2/H3 自动生成，滚动监听高亮
- [ ] **评论区**：基于 localStorage 的简单评论区（可扩展到后端）
- [ ] **阅读进度条**：文章页顶部进度条
- [ ] **上/下篇文章导航**

### 文章管理（Electron 模式）
- [ ] 内置 Markdown 编辑器（编辑/预览分屏）
- [ ] 文章增删改操作
- [ ] 图片拖拽上传到 `assets/` 目录
- [ ] 一键发布到博客

### 懒加载 & 性能
- [ ] 图片懒加载
- [ ] 首屏 CSS 内联
- [ ] 文章列表虚拟滚动（如果文章超过 50 篇）

## 6. 技术方案

### 前端
- **HTML5** + **CSS3**（自定义属性变量）+ **Vanilla JS**（ES6+）
- Markdown 解析：marked.js
- 代码高亮：highlight.js
- 图标：Lucide Icons (CDN)

### 文章数据
- 文章以 `.md` 文件存在 `articles/` 目录
- 元信息（标题、日期、标签、封面图）在文章开头的 YAML front-matter
- `articles/index.json` 索引文件（自动生成/手动维护）

### Electron 打包
- 主进程：窗口管理、系统托盘、文件读写
- 渲染进程：博客前端
- 打包工具：electron-builder → `.exe`

### 文章数据格式 (front-matter)
```yaml
---
title: "文章标题"
date: 2026-04-08
tags: ["技术", "前端"]
cover: "/assets/cover-xxx.jpg"
excerpt: "文章摘要..."
---
```
