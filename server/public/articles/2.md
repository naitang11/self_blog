# 用 Vanilla JS 搭博客：比框架更自由的体验

React/Vue 很强大，但有时候就是想用最原始的方式做点东西。这篇讲讲我是怎么用纯 HTML + CSS + JS 搞定一个完整博客的，包括 Markdown 渲染、代码高亮、搜索和深色模式。

## 为什么放弃框架

不是说框架不好。Vue 和 React 我每天都在用，工作里也写了不少。但对于个人博客这种内容为主的项目，引入整个打包流程有点杀鸡用牛刀。

而且用框架意味着：版本升级要跟、依赖要维护、打包配置要管理。有时候只是想改一行字，结果 npm 报一堆警告，很烦。

Vanilla JS 的方式：**文件即页面，改完直接刷新看效果**，调试全靠浏览器 DevTools，简单直接。

## Markdown 渲染

核心就靠 [marked.js](https://marked.js.org/)：

```html
<script src="https://cdn.jsdelivr.net/npm/marked/marked.min.js"></script>
<script>
  document.getElementById('content').innerHTML =
    marked.parse(content);
</script>
```

文章存在 `.md` 文件里，用 `fetch()` 拉回来直接渲染。这么简单的东西，框架里往往要配半天。

## 代码高亮

用 Prism.js，配上常用的语言支持：

```html
<link rel="stylesheet" href="prism.css">
<script src="prism.js"></script>
```

再给 marked 加个 highlight 选项：

```javascript
marked.setOptions({
  highlight: function(code, lang) {
    return Prism.highlight(code,
      Prism.languages[lang] || Prism.languages.plaintext,
      lang
    );
  }
});
```

效果不比 GitHub 差，体积还小。

## 搜索功能

不用后端，直接在前端跑。思路很简单：

1. 加载时把文章标题/摘要/标签读进内存
2. 用户输入时做字符串匹配
3. 实时显示结果列表

```javascript
const results = ALL_ARTICLES.filter(a =>
  a.title.toLowerCase().includes(query) ||
  a.excerpt.toLowerCase().includes(query)
);
```

100 篇文章以内毫秒级响应，够用了。

## 深色模式

CSS 变量一把梭：

```css
:root {
  --bg: #FAFBFC;
  --text: #1A1D21;
}

[data-theme="dark"] {
  --bg: #0F1117;
  --text: #E8ECF0;
}

body { background: var(--bg); color: var(--text); }
```

JS 负责切换和数据持久化。记住用户偏好，下次打开还是他选的模式。

## 部署

就一堆静态文件。扔到 Vercel、Netlify、Cloudflare Pages 都行，或者直接拷到服务器 nginx 目录。不用 build，不用跑服务，FTP 都能部署。

## 总结

工具没有最好的，只有最适合的。对于一个以内容为核心的个人博客，Vanilla JS 完全能 hold 住，而且维护成本极低。下次想加功能，5 分钟改完，不用等 CI 跑半天。
